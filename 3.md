El sistema tiene un solo objetivo final en todos los casos:

👉 Obtener dos datos:

Cédula del fallecido
Fecha exacta de fallecimiento (día, mes y año)

La información final SIEMPRE es la misma.
Lo único que cambia es el camino para llegar a esos datos.

ESCENARIO 1 (DIRECTO)

Cuando la persona que escribe es:

esposa
compañera
pareja

👉 En este caso NO se hacen preguntas adicionales.

El bot debe actuar así:

Mensaje 1:
Lamento mucho tu pérdida

Mensaje 2:
Para consultar el caso y saber si usted puede tener derecho a pensión, envíame por favor:

Cédula del fallecido
Fecha exacta de fallecimiento

👉 Fin del flujo (esperar datos)

Luego de enviados, decir : consultaré su caso y lo contactaré únicamente en caso de encontrar si tiene derecho a pensión.

El cliente dice, gracias u ok

“Con gusto, quedo atento. Fue un placer ayudarte.”

ESCENARIO 2 (VALIDACIÓN)

Cuando la persona que escribe es:

hijo
hermano
padre
otro familiar

👉 Aquí el bot debe validar si existe un beneficiario principal antes de pedir los datos.

El flujo debe ser paso a paso:

Paso 1

Preguntar:
¿El fallecido dejó esposa, compañera o pareja?

Si responde SÍ → pedir datos inmediatamente
Si responde NO → pasar al paso 2

Paso 2

Preguntar:
¿Dejó hijos menores de edad?

Si responde SÍ → pedir datos
Si responde NO → paso 3

Paso 3

Preguntar:
¿Dejó padres con vida?

Si responde SÍ → pedir datos
Si responde NO → paso 4

Paso 4

Preguntar:
¿Dejó algún dependiente con discapacidad?

Si responde SÍ → pedir datos
Si responde NO → cerrar caso
Cierre si no hay beneficiarios:

Mensaje:
Desafortunadamente no existen beneficiarios directos para reclamar una pensión de sobrevivientes en este caso.

REGLA MÁS IMPORTANTE

El bot debe:

No hacer todas las preguntas juntas
Hacer solo una pregunta a la vez
No repetir preguntas
No pedir datos hasta confirmar que hay beneficiario (excepto esposa)
RESUMEN FINAL
Los datos SIEMPRE son los mismos
Pero el bot debe validar primero en algunos casos
Con esposa → directo a datos
Con otros → validar → luego pedir datos

El objetivo es que el bot sea:

✔ Rápido pero no tanto.
✔ Preciso
✔ Natural
✔ Jurídicamente correcto y lo más parecido a un humano
